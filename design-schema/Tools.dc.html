07 IN THE REGISTRY
01

Metric Screener

Client-side app

A validation engine that tests EHS performance metrics for predictive validity. Running lag-correlation analysis on monthly uploads, it algorithmically separates true leading indicators — those giving operational signal before an event — from compliance metrics that merely count what already went wrong.

02

EHS LangExtract Prototypes

Python · LLM

A suite of targeted extraction frameworks that eliminate administrative friction in safety workflows. Using schema-constrained LLM parsing, they automate the conversion of chaotic EHS text — incident narratives, SDS data, regulatory clauses — into audit-ready JSON payloads. The technical blueprint for moving from manual entry to automated data refineries.

03

SWA Behavioral Dynamics Simulator

Simulation

An interactive behavioral-economics engine modeling Stop Work Authority decisions. Rejecting the 'rational worker' assumption, it applies Prospect Theory and Hyperbolic Discounting to calculate why field crews avoid using SWA — mapping safety culture as a quantifiable algorithm rather than a lagging observation.

04

Safety Identity Analysis

NLP Model

An NLP pipeline decoding the professional identity of the safety industry. Ingesting unstructured practitioner responses, it uses SBERT embeddings, K-Means clustering, and UMAP to map semantic archetypes — quantifying the gap between compliance metrics (ISO 45001) and operational resilience (Safety-II) via verb classification.

05

Regulens for OSHA

Chrome Extension

A production Chrome extension that modifies the osha.gov DOM to eliminate information friction. It detects and embeds official OSHA Letters of Interpretation directly beside the target standard, and maps regulatory cross-references (e.g. 29 CFR 1910.134) in real time — tightening the loop between monolithic compliance text and operational queries.

06

OSHA Incident Classification Model

Streamlit · ML

An ML pipeline evaluating Logistic Regression, XGBoost, and BERT to classify unstructured injury narratives. Trained on a decade of OSHA Severe Injury Reports (2015–2025), it parses extreme events against the OIICS v2.01 standard (Nature, Part, Source, Event) — demonstrating how to pull quantitative signal from free-text field data.

View Application →
07

Dropped Object Assessment

iOS App

A native iOS engine for field engineers calculating dropped-object risk (DROPS). It computes potential impact energy in Joules from mass and drop height, mapping the output to structural risk classifications — replacing spreadsheet field notes with standardized assessment data and compliance-ready PDF reports generated on-site.
